- Batch multiple tool calls per step to cut round trips.
- Keep tool output lean; pipe through grep/head when possible.
- Prefer project-specific runner scripts over raw build/test commands.
- Use find-src to locate source files (single or comma-separated extensions) instead of raw find. Add --count to tally files.
- Use read-lines to inspect file sections or read multiple files at once. Add --numbered/-n for line-numbered output.
- Use search-src instead of raw grep --include (unsupported on busybox). Specify extension to limit scope.
- Use search-src --files-only to list filenames by content and --head=N to limit results.
- Use edit-file for safe in-place edits: insert lines before/after a pattern, replace text, or use replace-block for multi-line block replacements.
- Use run-test to run tests with concise PASS/FAIL output instead of full verbose logs.
- Avoid cd to non-existent directories; verify paths with ls first.
- Combine multiple independent reads or searches in one step.
