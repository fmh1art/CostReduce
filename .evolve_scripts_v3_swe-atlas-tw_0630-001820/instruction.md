# Cost-saving instructions

- Batch operations: batch_read f1 f2 f3, find_in_files dir p1 p2 --include=*.ext, multi_replace f old1 new1 old2 new2. STOP-PROGRESSING-IF making N separate calls when N can fit in one.
- batch_read: --cd=DIR, --grep=P, --max-results=N (limit grep matches), --context=N/-A N/-B N, --count-lines, file:N-M/file:N-, --find=GLOB, --head=N, --number, --brief. STOP-PROGRESSING-IF using nl -ba file | sed -n 'X,Yp' -- use batch_read file:X-Y; STOP-PROGRESSING-IF using cat, head, tail, wc -l, cat -A, sed -n, find ... -exec cat, grep -n on files, or cat|grep|head pipelines -- use batch_read --grep --max-results.
- find_in_files --names-only for file discovery, --ls for dir listing, --path/--name-regex for path filtering, -l for filename-only content search. STOP-PROGRESSING-IF using find|head, find|xargs grep, grep -rn --include, or ls -la.
- code_structure --summary for definition line numbers, then batch_read file:N-M; --py-inspect --cd=DIR file.py [names]. STOP-PROGRESSING-IF grepping for nonexistent functions.
- multi_replace for all file edits (replace, insert-before/after, insert-at-line, delete-matching, append, prepend). NEVER sed -i. STOP-PROGRESSING-IF using sed -i for config edits or cat > file << 'EOF' to rewrite files (use write_file instead).
- run_cmd --dir=DIR --venv=PATH --pythonpath=PATH --env=KEY=val (replaces cd+source+export chains). py_exec --cd=DIR --venv=PATH --pythonpath=PATH --env=KEY=val for short inline scripts. STOP-PROGRESSING-IF doing cd && . venv/bin/activate && export KEY=val && python -c "..." -- use py_exec --cd --venv --pythonpath --env instead.
- py_exec --cd=DIR --venv=PATH --pythonpath=PATH for venv activation and PYTHONPATH, --node for Node.js, -f - for heredoc stdin scripts. STOP-PROGRESSING-IF writing scripts to /tmp and executing separately.
- run_tests --jest/--pytest/--go --dir=DIR --tz=TZ --summary --failed --tail=N --grep=P --xvfbrun --env=KEY=val. Auto-activates venv in /app/venv. STOP-PROGRESSING-IF using export PATH=... && cd ... && go test/vet/build chains; STOP-PROGRESSING-IF using cd&&npx jest or cd&&python -m pytest.
- quick_map instead of multiple ls/find/tree calls; --filter=GLOBS for file types.
- read_json file key1 key2 --filter=S instead of cat package.json|python3 -c pipelines.
- write_file path content for file creation; --append for appending (replaces cat >> file << 'EOF').
- ensure_services postgresql redis before tests; --reset for stale pid; --reinit for corrupted data dirs.
- build_check for Go (auto-sets CGO_ENABLED=0), --ts for TypeScript with flags. STOP-PROGRESSING-IF using npx tsc --noEmit or export CGO_ENABLED=0 go build chains.
- git_diff --checkout=FILE to reset modified files. STOP-PROGRESSING-IF using git checkout -- file.
- psql -c "SQL" (defaults test/test@localhost:5432/test), --db/--user/--pass/--host/--port, -f files, --grep, --head. STOP-PROGRESSING-IF typing PGPASSWORD=... psql -U test -h localhost; STOP-PROGRESSING-IF using python -c with SQLAlchemy for simple SQL queries -- use psql instead.
- STOP-PROGRESSING-IF rereading files whose content is already known; STOP-PROGRESSING-IF calling scripts as ./name/ not /app/.preinstalled_scripts/name/main.sh.
- STOP-PROGRESSING-IF node -e "require('...')" gets MODULE_NOT_FOUND -- check dist/ with batch_read --find=GLOB, verify package.json with read_json.
- Use batch_read --brief to suppress file headers when only file content is needed (saves observation token cost).
- STOP-PROGRESSING-IF chaining cd && . venv/bin/activate && CONFIG=... DB_URI=... FLASK_ENV=... PYTHONPATH=/app python -c "..." -- use py_exec --cd --venv --pythonpath --env multiple times instead.
- STOP-PROGRESSING-IF doing multiple sequential psql -c "SQL" calls that could be combined with semicolons into one -c call.
- STOP-PROGRESSING-IF cat package.json then grep -E on output -- use batch_read --grep with --max-results on the json file directly.
